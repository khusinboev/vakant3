## AUTH FLOW — Telegram Login Widget

```
1. User opens webapp → sees Login page
2. Clicks "Telegram orqali kirish" → Telegram Login Widget popup
3. Widget returns: { id, first_name, username, photo_url, auth_date, hash }
4. Frontend POST /api/auth/telegram { tg_data: {...} }
5. Backend verifies HMAC-SHA256 with bot TOKEN
6. Backend upserts user into `users` table (saves first_name, username, photo_url)
7. Creates row in webapp_sessions (token = secrets.token_urlsafe(32), expires 30d)
8. Returns { session_token, user }
9. Frontend saves token to localStorage, stores user in zustand
```

### telegram_auth.py
```python
import hashlib, hmac, time
from typing import Any

def verify_telegram_login(data: dict[str, Any], bot_token: str) -> bool:
    """Telegram Login Widget HMAC-SHA256 verification."""
    check_hash = data.pop("hash", "")
    if not check_hash:
        return False
    
    # auth_date freshness check (max 1 hour)
    auth_date = int(data.get("auth_date", 0))
    if time.time() - auth_date > 3600:
        return False
    
    data_check_string = "\n".join(
        f"{k}={v}" for k, v in sorted(data.items())
    )
    secret_key = hashlib.sha256(bot_token.encode()).digest()
    computed = hmac.new(secret_key, data_check_string.encode(), hashlib.sha256).hexdigest()
    return hmac.compare_digest(computed, check_hash)
```

## BACKEND ENDPOINTS SPEC

### Auth
```
POST /api/auth/telegram
  body: { id, first_name, username?, photo_url?, auth_date, hash }
  → verifies Telegram hash
  → upserts user (UPDATE first_name, username, photo_url)
  → creates session
  returns: { session_token: str, user: UserProfile }

POST /api/auth/logout
  header: Authorization: Bearer <token>
  → deletes session from webapp_sessions
  returns: { ok: true }

GET /api/auth/me
  returns: UserProfile | 401
```

### Jobs
```
GET /api/jobs/search?page=1&money=0&region_soato=&district_soato=&specs=
  → proxies to osonish.uz fetch_osonish_list()
  → uses same cache layer (make_cache_key + cache_get/cache_set)
  returns: {
    vacancies: [VacancyItem],
    page: int,
    last_page: int,
    total_estimate: int
  }

VacancyItem: {
  uid: str,          # "osonish_12345"
  title: str,
  company: str,
  salary_text: str,
  location: str,
  district: str,
  posted_at: str,
  is_saved: bool     # requires auth, false if not logged in
}

GET /api/jobs/{uid}
  → fetches detail from osonish.uz (fetch_osonish_detail)
  → cached 1h
  returns: VacancyDetail (full fields: info, address, hr_phone, etc.)
```

### Saves
```
GET /api/saves?page=1&limit=10
  auth required
  → JOIN saves + osonish detail (from cache if available)
  returns: { items: [VacancyDetail], total: int }

POST /api/saves/{uid}
  auth required
  → INSERT OR IGNORE INTO saves
  returns: { saved: true }

DELETE /api/saves/{uid}
  auth required
  → DELETE FROM saves WHERE user_id=? AND save_id=?
  returns: { removed: true }
```

### Profile
```
GET /api/profile
  auth required
  returns: {
    user: UserProfile,
    stats: {
      saves_count: int,
      referrals_count: int,
      member_since: str,      # formatted date
      days_active: int
    },
    current_filters: {
      region: str | null,
      district: str | null,
      specs: str | null,
      money: int | null
    }
  }

UserProfile: {
  user_id: int,
  first_name: str,
  username: str | null,
  photo_url: str | null,
  lang: str
}

PATCH /api/profile/filters
  auth required
  body: { region?: str, district?: str, specs?: str, money?: int }
  → UPDATE users SET ... WHERE user_id = ?
  → invalidates search cache for this user
  returns: { updated: true }
```

## UX REQUIREMENTS

### Mobile-first
- Bottom navigation bar: 🏠 Bosh sahifa | ❤ Saqlangan | 👤 Profil
- All modals → bottom drawers on mobile (not centered modals)
- Touch-friendly tap targets (min 44px)
- Pull-to-refresh on job list

### Language
- All UI text in **Uzbek** (same as bot)
- Numbers: `1 000 000 so'm` format (space separator)
- Dates: `DD.MM.YYYY` format (same as bot)

### Performance
- Search results: React Query with `keepPreviousData: true` (no flicker on page change)
- Vacancy detail: prefetch on card hover (desktop)
- Regions/districts/specs: cached in React Query for 24h (staleTime)
- Images: Telegram photo_url with fallback to initials avatar

### Error states
- osonish.uz timeout → "Natijalar yuklanmadi. Qayta urinib ko'ring"
- Network error → toast notification
- 401 on any request → clear session + redirect /login

## SECURITY
- Session token: `secrets.token_urlsafe(32)`, stored in DB, 30-day expiry
- Telegram hash verification: mandatory on every login attempt
- Rate limits (slowapi):
  - POST /api/auth/telegram → 10/minute per IP
  - GET /api/jobs/search → 30/minute per IP
  - POST /api/saves/* → 60/minute per user
- CORS: configurable `WEBAPP_ORIGIN` env var (default `http://localhost:5173`)
- No sensitive data in responses (no other users' user_ids exposed)
